## 2. 任务类型与后台判定

## 2.1 `tasks/types.ts`

已核对要点：

1. `TaskState` 是多个具体任务状态的联合（local shell/agent/remote 等）。
2. `isBackgroundTask` 判定要求：
   - 状态是 `running/pending`
   - 且若有 `isBackgrounded` 字段则不能为 `false`（前台任务不计入后台指示器）。

## 3. 通用停止链路

## 3.1 `tasks/stopTask.ts`

已核对要点：

1. 统一 stop 入口用于 TaskStopTool 和 SDK stop_task 请求。
2. 失败原因显式分类：`not_found/not_running/unsupported_type`。
3. local shell 停止后有“抑制噪声通知 + 补发 SDK terminated 事件”的专门处理。

## 4. LocalShell 任务语义

## 4.1 `tasks/LocalShellTask/LocalShellTask.tsx`

已核对要点：

1. LocalShellTask 状态初始化包含 `command/shellCommand/isBackgrounded` 等字段。
2. 背景执行有 stall watchdog：长时间无增长且 tail 命中交互提示模式时发 task-notification。
3. 完成路径会区分 `completed/failed/killed`，并写入统一 XML 标签通知。
4. 任务结束后触发输出淘汰（`evictTaskOutput`）避免内存长期持有。

---

## 5. LocalAgent 任务语义

## 5.1 `tasks/LocalAgentTask/LocalAgentTask.tsx`

已核对要点：

1. 状态包含 `progress/messages/pendingMessages/retain/diskLoaded/evictAfter` 等面板协同字段。
2. `queuePendingMessage + drainPendingMessages` 支持工具轮次边界上的消息注入。
3. 通知发送前会原子检查并设置 `notified`，避免重复推送。
4. kill 路径会 abort controller、清理 cleanup，并根据 retain 设置可见性/淘汰期限。

---

## 6. RemoteAgent 任务语义

## 6.1 `tasks/RemoteAgentTask/RemoteAgentTask.tsx`

已核对要点：

1. 状态含 `sessionId/remoteTaskType/remoteTaskMetadata/log/todoList` 等远程协作信息。
2. 有远程任务类型集合（remote-agent/ultraplan/ultrareview/autofix-pr/background-pr）。
3. 提供 completion checker 注册机制，支持按任务类型外挂完成判定。
4. 远程通知同样采用“先原子置 notified，再入队消息”策略。
5. 针对 ultraplan/review 有专门 tag 抽取与失败通知分支。

## 7. 框架层：注册、轮询、通知与淘汰

## 7.1 `utils/task/framework.ts`

已核对要点：

1. `registerTask` 处理首次注册与 resume 替换场景（保留部分 UI 状态）。
2. `updateTaskState` 对 no-op 引用返回做优化，避免无效 re-render。
3. 轮询主逻辑：
   - `generateTaskAttachments` 读取增量输出并收集 offset patch/evict 列表
   - `applyTaskOffsetsAndEvictions` 在 fresh state 上合并，避免并发状态回退
4. 任务通知消息统一包含 `task_id/task_type/output_file/status/summary` 等 XML 字段。
5. `POLL_INTERVAL_MS/STOPPED_DISPLAY_MS/PANEL_GRACE_MS` 体现任务展示生命周期策略。

---

## 8. 输出落盘与读取链路

## 8.1 `utils/task/diskOutput.ts`

已核对要点：

1. 任务输出目录是 `project temp + sessionId + tasks`，并对 sessionId 首次调用做缓存。
2. 写入使用 `DiskTaskOutput` 队列化排空，强调内存回收与避免大队列闭包滞留。
3. Unix 使用 `O_NOFOLLOW` 防符号链接跟随，降低宿主文件写入攻击面。
4. 有输出硬上限（`MAX_TASK_OUTPUT_BYTES=5GB`）与截断提示。
5. 提供 `append/flush/evict/getTaskOutputDelta/getTaskOutputPath` 组成完整输出链。

---

## 8.2 `utils/task/outputFormatting.ts`

已核对要点：

1. API 侧输出长度受 `TASK_MAX_OUTPUT_LENGTH`（默认 32k，上限 160k）控制。
2. 超限时返回尾部截断内容，并附“完整输出文件路径”头信息。

---

## 9. 任务子系统不变量（从实现归纳）

1. 所有通知发送前都应先做 notified 原子置位，避免重复事件。
2. 输出读取应以增量偏移推进，不应每轮全量读文件。
3. 任务状态更新需基于 fresh state 合并，避免 await 后状态回滚。
4. 终态任务需结合 retain/grace 条件决定是否即时淘汰。
