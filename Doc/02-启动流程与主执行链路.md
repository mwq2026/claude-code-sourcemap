# 02｜启动流程与主执行链路

## 1. 启动前置：`main.tsx` 顶层副作用

`main.tsx` 顶部有非常明确的启动注释，强调三个必须最早执行的副作用：

1. `profileCheckpoint('main_tsx_entry')`：记录入口性能节点
2. `startMdmRawRead()`：并行触发 MDM 读取（减少后续阻塞）
3. `startKeychainPrefetch()`：并行预取 keychain（避免串行读取导致启动慢）

这是典型的“**利用模块加载窗口做预热并行**”策略。

## 2. 初始化主链：`main.tsx` → `entrypoints/init.ts`

入口在 `main.tsx` 中调用 `init()`（`memoize(async () => ...)`），初始化核心动作包括：

- `enableConfigs()` 启用配置系统
- `applySafeConfigEnvironmentVariables()` 先应用“安全环境变量”
- `setupGracefulShutdown()` 注册优雅退出
- 预初始化 remote managed settings 与 policy limits 的 loading promise
- `configureGlobalMTLS()` / `configureGlobalAgents()` 配置网络层
- `preconnectAnthropicApi()` 预连接 API 减少首请求延迟
- 按环境决定是否启用 upstream proxy

### 2.1 两阶段环境变量应用

- **阶段 1（init）**：仅应用 safe env（信任前）
- **阶段 2（interactiveHelpers）**：TrustDialog 后 `applyConfigEnvironmentVariables()` 应用完整 env

这体现了**工作区信任边界**设计：在信任未建立前，避免加载潜在危险配置。

## 3. 交互入口：`interactiveHelpers.tsx`

关键职责：

- `showSetupScreens()`：引导/信任/安全提示等对话流程
- `renderAndRun()`：统一渲染并等待退出
- `exitWithMessage/exitWithError()`：在 Ink 场景下安全输出并退出

### 3.1 setup 阶段核心顺序

1. Onboarding（首次）
2. TrustDialog（交互会话中始终看作信任边界）
3. 重置并初始化 GrowthBook
4. 预热系统上下文
5. MCP approval / CLAUDE.md 外链告警
6. 应用完整环境变量
7. 触发 telemetry 初始化

## 4. REPL 挂载：`replLauncher.tsx`

`launchRepl()` 的逻辑非常简洁：

- 动态 import `components/App.js` 与 `screens/REPL.js`
- 调用 `renderAndRun(root, <App><REPL /></App>)`

这意味着：

- REPL UI 不是入口即硬加载；
- UI 与入口逻辑解耦明显，便于无头模式复用核心能力。

## 5. 上下文注入：`context.ts`

### 5.1 `getSystemContext()`

主要产出：

- git 状态快照（分支、默认主分支、status、recent commits）
- 可选 cache breaker（feature gate 控制）

`getGitStatus()` 采用并发执行：

- `getBranch()`、`getDefaultBranch()`
- `git status --short`
- `git log --oneline -n 5`
- `git config user.name`

并做长度截断（2k）防止 prompt 膨胀。

### 5.2 `getUserContext()`

主要产出：

- `claudeMd` 内容（可禁用或 bare 模式下按条件跳过）
- `currentDate`（当前日期注入）

并把 CLAUDE.md 缓存进 bootstrap state，供分类器等模块读取，避免循环依赖。

## 6. 主执行路径（简化）

```text
CLI 入口 main.tsx
  -> init()
  -> 解析参数 + feature gate 路由
  -> getCommands()/getTools()
  -> 命令模式：直接执行命令 handler
  -> REPL 模式：
       launchRepl()
         -> App + REPL 挂载
         -> query.ts / QueryEngine.ts 执行消息循环
```

## 7. 设计要点总结

1. **启动并行化**：大量“早触发、晚 await”策略
2. **信任分层**：safe env 与 full env 分阶段
3. **动态加载**：大模块按需加载降低冷启动成本
4. **上下文可控注入**：严格限制上下文体积并缓存
5. **入口兼容多模式**：命令行/REPL/远程/assistant 共存
