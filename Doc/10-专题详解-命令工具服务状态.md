# 10｜专题详解：命令 / 工具 / 服务 / 状态

> 目标读者：学习者 + 一线研发  
> 目标：把“看起来分散的目录”组织成可执行的专题学习地图，并给出细化观察点。

---

## 1. 命令系统专题（`commands/`）

## 1.1 组织方式

- 聚合入口：`restored-src/src/commands.ts`
- 子域化目录：如 `plugin/`、`install-github-app/`、`mcp/`、`review/`、`context/`、`session/` 等
- 统计快照：`commands/` 约 207 文件

## 1.2 核心机制

1. 命令注册表按 feature/env/用户类型做可见性裁剪
2. 通过 memoize 延迟求值，减少启动期无谓加载
3. 命令层承接“用户意图入口”，并把执行委托给 tools/services

## 1.3 阅读建议

先读：

1. `commands.ts`（全局注册策略）
2. 高频命令域：
   - `commands/review/*`
   - `commands/mcp/*`
   - `commands/plugin/*`
   - `commands/config*` / `commands/model*` / `commands/session*`

重点关注：

- 命令是直接执行，还是转入 query/tool 路径
- 命令参数如何绑定权限/上下文/远程行为
- 命令输出是 UI 渲染还是结构化消息

## 2. 工具系统专题（`tools/`）

## 2.1 组织方式

- 聚合入口：`restored-src/src/tools.ts`
- 协议中心：`restored-src/src/Tool.ts`
- 统计快照：`tools/` 约 184 文件

高密度子域（当前快照）：

- `AgentTool/`（20）
- `BashTool/`（18）
- `PowerShellTool/`（14）
- `LSPTool/`、`FileEditTool/`、`WebFetchTool/`、`WebSearchTool/`、`MCPTool/` 等

## 2.2 工具定义共同结构（抽象）

一类工具通常包含：

1. `name/description`
2. `inputSchema`（运行时校验）
3. 执行逻辑（含 `tool_result` 内容）
4. 并发声明（`isConcurrencySafe`）
5. 中断语义（`interruptBehavior`）
6. 可选 UI 组件（`UI.tsx`）

## 2.3 高风险工具观察框架（研发建议）

针对 Bash/PowerShell/FileEdit 类工具：

1. 输入约束：路径、命令、模式是否合法
2. 权限前置：是否可被 deny/ask/allow 规则影响
3. 中断策略：用户中断时 cancel 还是 block
4. 错误语义：是否可稳定回填 `tool_result`，避免协议断裂

## 6. 远程协同专题（`remote/` + `bridge/` + `teleport`）

## 6.1 Remote manager

- `RemoteSessionManager`：会话总线管理、控制消息分流、用户事件发送
- `SessionsWebSocket`：订阅、重连、关闭码策略、控制响应发送

## 6.2 报文与控制面

1. 事件发送：HTTP `/v1/sessions/{id}/events`
2. 会话消息：WS subscribe 流
3. 控制协议：`control_request/response/cancel`

## 6.3 排障必查键

- `request_id`、`tool_use_id`、`uuid`、`session_id`、`task_id`

## 7. 研发场景下的“改动影响面”模板

当你修改某一子系统时，建议按下表快速评估：

1. **接口面**：是否改动了 Tool/SDK/control schema
2. **状态面**：是否新增全局状态字段或高频更新字段
3. **权限面**：是否改变了 canUseTool/deny rule/remote permission 行为
4. **时序面**：是否改变了 stream 解析顺序或 tool 回填顺序
5. **容错面**：异常路径是否仍能产出闭环消息
6. **gate 面**：feature 组合是否引入新分支风险
