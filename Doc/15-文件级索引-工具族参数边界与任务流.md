## 2. 文件读写工具族

## 2.1 `tools/FileReadTool/FileReadTool.ts`

已核对要点：

1. 输入参数核心为 `file_path`，支持 `offset/limit`（文本分段）与 `pages`（PDF 页范围）。
2. 输出是判别联合：`text/image/notebook/pdf/parts/file_unchanged`，并非单一文本返回。
3. 有设备文件阻断列表（如 `/dev/zero`、`/dev/random`、`/dev/stdin`、`/dev/tty` 等），避免挂死/无限输出。
4. 包含对图片、PDF、notebook 的专用处理链，且有 token/大小相关限制与提示。
5. 具备 read permission 检查与 deny 规则联动（通过文件系统权限上下文）。

---

## 2.2 `tools/FileWriteTool/FileWriteTool.ts`

已核对要点：

1. 输入为 `file_path + content`，严格 schema。
2. `backfillObservableInput` 会对 `file_path` 做 `expandPath`，避免 `~` / 相对路径绕过 allowlist。
3. `validateInput` 包含多层防护：
   - team memory secret guard
   - deny rule 检查
   - UNC 路径特殊处理（规避 NTLM 凭据泄露风险）
   - “先读后写 + mtime 校验”防并发写冲突
4. 写前后联动技能发现/激活、file history、LSP 变更通知、诊断清理。
5. 写入是全量内容替换，保留模型给定换行语义，不做隐式行尾改写。

---

## 3. 搜索发现工具族

## 3.1 `tools/GrepTool/GrepTool.ts`

已核对要点：

1. 支持 `output_mode=content/files_with_matches/count` 三种输出形态。
2. 支持 `head_limit + offset` 分页截断语义，并在结果中显式回传应用的分页参数。
3. 默认排除 VCS 目录（`.git/.svn/.hg/.bzr/.jj/.sl`）降低噪声。
4. `multiline` 需显式开启才附加 `-U --multiline-dotall`。
5. 路径输入校验与 UNC 路径安全保护逻辑齐全，权限检查走 read permission。

---

## 3.2 `tools/GlobTool/GlobTool.ts`

已核对要点：

1. 输入是 `pattern` 与可选 `path`，`path` 省略时默认 cwd。
2. `validateInput` 会校验 path 存在且为目录；不存在时给 cwd 下候选建议。
3. 调用阶段有结果数量上限（默认/配置 `limit`）与 `truncated` 回传。
4. 输出路径会转相对路径（cwd 下），节省 token。
5. 工具被标记为并发安全 + 只读。

---

## 4. 网络检索工具族

## 4.1 `tools/WebFetchTool/WebFetchTool.ts`

已核对要点：

1. 输入是 `url + prompt`，先做 URL 有效性校验。
2. 权限检查基于域名规则内容（`domain:<hostname>`）做 allow/ask/deny 决策，并支持预批准 host 快速放行。
3. 遇到跨 host 重定向会返回“需用重定向 URL 重试”的结构化提示，而不是静默跟随跨域。
4. 对预批准 + markdown + 长度阈值内场景可直接返回内容；否则走 `applyPromptToMarkdown` 摘要链。
5. 二进制内容可落盘并在结果追加“保存路径”提示，便于后续 Read 工具跟进。

---

## 4.2 `tools/WebSearchTool/WebSearchTool.ts`

已核对要点：

1. 输入支持 `query` 与 `allowed_domains/blocked_domains`（两者互斥）。
2. `isEnabled` 受 provider + model 能力组合约束（firstParty/vertex/foundry 分支）。
3. 调用通过模型流事件处理 `server_tool_use` 与 `web_search_tool_result`，把搜索命中与文本说明组装成统一输出。
4. 工具 schema 固定 `max_uses: 8`。
5. 权限返回 `passthrough` 并附建议规则，交由上层权限链路处理。

## 5. 任务流工具族（Todo/后台任务）

## 5.1 `tools/TaskCreateTool/TaskCreateTool.ts`

已核对要点：

1. 创建任务后会执行 task-created hooks；若出现 blocking error，会回滚删除新任务。
2. 任务创建成功会自动展开任务视图（`expandedView='tasks'`）。
3. 仅在 `isTodoV2Enabled()` 时启用。

---

## 5.2 `tools/TaskGetTool/TaskGetTool.ts` 与 `TaskListTool.ts`

已核对要点：

1. `TaskGet` 提供单任务读取；不存在返回 `task: null`。
2. `TaskList` 会过滤内部任务（`metadata._internal`）。
3. `TaskList` 输出时会把已 completed 任务从 `blockedBy` 展示集合中剔除，减少噪声依赖。

---

## 5.3 `tools/TaskStopTool/TaskStopTool.ts`

已核对要点：

1. 主参数是 `task_id`，兼容老参数 `shell_id`（用于旧 KillShell 兼容）。
2. `validateInput` 明确检查任务存在且状态为 `running`。
3. `aliases: ['KillShell']` 保持历史 transcript/SDK 兼容。

---

## 5.4 `tools/TaskOutputTool/TaskOutputTool.tsx`

已核对要点：

1. 工具已标注 deprecated，描述与 prompt 均建议“优先 Read 任务输出文件”。
2. 支持 `block=true/false` 与 `timeout`：
   - 非阻塞返回 `not_ready/success`
   - 阻塞轮询等待并返回 `success/timeout`
3. 对不同任务类型（local_bash/local_agent/remote_agent）有差异化输出拼装。
4. 完成后会将任务标记 `notified: true`，避免重复通知语义混乱。

## 6. 参数边界与常见误用（跨工具）

1. **路径默认值误传**：`Glob.path` 应省略而非传 `"undefined"` / `"null"`。
2. **读后写约束**：`FileWrite` 在已有文件场景依赖“先读 + 时间戳一致”，否则拒绝写入。
3. **域名权限误判**：`WebFetch` 权限规则按 hostname 映射，不是按完整 URL 文本匹配。
4. **分页遗漏**：`Grep` 大结果应使用 `head_limit + offset`，否则易造成上下文膨胀。
5. **弃用工具继续依赖**：`TaskOutputTool` 为兼容保留，新增流程应转向“任务输出文件 + Read”路径。
